# 依赖与目标框架兼容策略

## 目标框架基线
- App：`net8.0-windows`
- Core / Tests：`net8.0`

该基线保证了运行时一致性：业务库与测试库共享 .NET 8 生态，WPF UI 保持 Windows 桌面能力。

## 版本治理规则
- 使用集中包版本管理：`Directory.Packages.props`。
- `Microsoft.Extensions.*` 统一锁定在 8.x，与目标框架主版本保持一致。
- 非框架库（如 `Serilog`、`SSH.NET`）按稳定版本维护，但需经过构建与测试验证后再升级。
- 禁止在单个 `.csproj` 中写散落版本号，避免版本漂移。

## 升级策略
- 小版本升级（补丁/次版本）：按月评估，出现安全修复时可插队升级。
- 大版本升级（例如 8 -> 9）：仅在目标框架升级计划内执行，并要求先完成兼容分支验证。
- 升级必须通过：
  1. `dotnet build`（Release）
  2. 全量测试
  3. 关键链路测试门禁
  4. 覆盖率阈值门禁

## CI 质量门禁
- 覆盖率阈值：总行覆盖率低于阈值即失败。
- 关键链路门禁：对 `RuleEngine`、配置加载、Socks 边界、上游路由连接等关键测试执行单独筛选运行。

## 变更约定
- 新增依赖时，先更新 `Directory.Packages.props`，再在项目文件中添加不带版本号的 `PackageReference`。
- 任何目标框架变更必须同步更新本文件与 CI 配置。
