# SSH Client 新一轮重构计划

> 本文件为新计划，不合并历史执行记录。
> 生成时间：2026-03-30

## 目标
- 提升退出流程稳定性，避免异步清理遗漏。
- 提升代理链路健壮性，修复边界条件异常。
- 收敛配置来源与模块职责，降低维护复杂度。
- 在不改变现有用户行为前提下，逐步增强性能与可测试性。

## 重构原则
- 先补测试，再改实现。
- 每次改动保持可编译、可运行、可回退。
- 优先处理高风险与高收益项。

## 分阶段计划

### 阶段 A：稳定性修复（高优先级）
- [x] 统一应用退出链路，移除 fire-and-forget 清理路径。
- [x] 降低强制 `Environment.Exit(0)` 触发概率，改为可观测超时兜底。
- [x] 修复 SOCKS 握手 EOF 边界处理（`nMethods <= 0` 防护）。
- [ ] 为退出流程和 SOCKS 边界补回归测试（已完成 SOCKS 边界回归，退出流程回归待补）。

### 阶段 B：配置与生命周期收敛
- [x] 明确并统一运行时配置来源（Host 配置与用户配置职责划分）。
- [x] 对配置加载失败和路径异常提供一致日志与降级行为。
- [x] 更新架构文档中的配置优先级与生效时机说明。

### 阶段 C：代理与规则引擎性能优化
- [x] 对规则模式进行预解析与缓存，减少热路径重复 Regex 构造。
- [x] 评估 RuleEngine 的匹配顺序与短路策略，补充性能基准。
- [x] 为高频代理流量场景补充压力测试样例。

### 阶段 D：能力与模型一致性
- [x] 清理未落地或未暴露能力（如未使用规则类型）或补齐入口。
- [x] 对 `StrictHostKeyChecking` 与 `JumpHosts` 做“实现或下线”决策。
- [x] 保持 UI 能力、配置模型、运行时行为三者一致。

### 阶段 E：工程治理与质量门禁
- [x] 清理模板遗留文件（如无意义占位类）。
- [x] 梳理依赖版本与目标框架兼容策略。
- [x] 在 CI 中补充覆盖率阈值与关键链路测试门禁。

### 阶段 F：性能问题检查（新增）
- [x] 完成现状核查：当前仓库暂无专门性能基准或压测用例，暂不能判定“性能优秀”。
- [x] 完成负载证据核查：当前仅有功能测试，尚不足以证明“负载足够高”。
- [x] 新增性能基线检查（Release）：规则匹配吞吐、连接建立时延、内存与 GC 采样。
- [x] 新增负载压测场景：并发 1/10/50/100，持续 10 分钟，统计成功率与错误率。
- [x] 输出性能报告：记录 P50/P95 延迟、吞吐、CPU、内存峰值，并形成可追踪基线。

## 验收标准
- `dotnet build` 无错误，且不引入新的告警。
- `dotnet test` 全量通过。
- 退出流程可重复验证，无挂起或强退。
- 关键代理链路在异常输入下行为可预期。
- 完成性能基线与压测报告后，再判定是否达到“性能优秀、负载充分”。
